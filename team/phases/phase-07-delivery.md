# Phase 7: 交付和说明

## 目标

确保用户能够理解和使用产出。

## STATE VARIABLES

| 变量名 | 描述 | 来源 |
|--------|------|------|
| `{acceptance_result}` | 验收结果 | Phase 6 |
| `{delivery_package}` | 交付包 | 本 Phase |
| `{user_guide}` | 用户使用指南 | 本 Phase |

## 触发条件

**所有模式必须执行**

Phase 6 用户验收通过后

## 执行步骤

### Step 1: 整合交付物

```yaml
交付物清单:
  技术产出:
    - 代码变更
    - 配置文件
    - 数据库迁移

  文档产出:
    - 用户使用指南
    - 关键决策说明
    - 注意事项
```

### Step 2: 生成用户使用指南

```markdown
# {功能名称} 使用指南

## 这个功能是什么
{用简单语言解释，不超过3句话}

## 如何使用
1. {步骤1}
2. {步骤2}
3. {步骤3}

## 注意事项
- {注意1}
- {注意2}

## 常见问题

### Q: {问题1}
A: {回答1}

### Q: {问题2}
A: {回答2}

## 需要帮助？
如果遇到问题，请联系 {contact}
```

### Step 3: 关键决策说明

向用户解释对用户有影响的决策：

```markdown
# 关键决策说明

## 做了什么决定
{决策描述}

## 为什么这样做
{原因，用简单语言}

## 对您的影响
{影响说明}

## 如果需要调整
{调整方法}
```

### Step 4: 优雅关闭团队

```javascript
// 通知所有成员任务完成
SendMessage({
  type: "broadcast",
  content: "任务已完成，感谢各位的贡献！",
  summary: "任务完成"
})

// 关闭团队
TeamDelete()
```

## 输出格式

### 用户交付包

```markdown
# 交付确认

## 交付内容

### 功能清单
| 功能 | 状态 | 说明 |
|------|------|------|
| {功能1} | ✅ 完成 | {说明} |
| {功能2} | ✅ 完成 | {说明} |

### 文件变更
| 文件 | 变更类型 | 说明 |
|------|----------|------|
| {file1} | 新增 | {说明} |
| {file2} | 修改 | {说明} |

### 使用指南
{user_guide 链接}

### 技术说明
{tech_notes 链接}

## 验收确认
- 验收人: {user}
- 验收时间: {timestamp}
- 验收结论: {result}

## 后续支持
如有问题，请通过以下方式联系:
- {contact_method}
```

```markdown
# 您的功能已就绪！

## 完成的功能

### 1. {功能名称}

### 2. {功能名称}

## 如何开始使用
1. {第一步}

## 如果遇到问题

- 或联系 {联系方式}

感谢您的信任！🎉
```
