# Interview Skills Test Cases

> 测试用例定义 - 需求访谈技能

---

## TC-INT-001: Feature 模式测试

**类型**: functional
**优先级**: P0

### 输入
```
/interview --mode feature
用户需求: 我想做一个用户登录功能
```

### 预期行为
1. 识别为 feature 模式
2. 开始多轮问题
3. 生成实施计划

### 验证点
- [ ] 正确识别模式
- [ ] 顺序提问（非并行）
- [ ] 生成实施计划文档

---

## TC-INT-002: 用户类型检测测试

**类型**: functional
**优先级**: P0

### 输入 A (技术用户)
```
我想实现一个基于 JWT 的 REST API 认证系统
```

### 输入 B (非技术用户)
```
我想要一个像微信那样的登录功能
```

### 预期行为
1. 正确检测用户类型
2. 非技术用户触发盲点模块

### 验证点
- [ ] 输入 A 识别为技术用户
- [ ] 输入 B 识别为非技术用户
- [ ] 非技术用户应用 11 个盲点问题

---

## TC-INT-003: 智能终止测试

**类型**: functional
**优先级**: P1

### 输入
```
/interview --mode feature
(经过多轮问答后)
```

### 预期行为
1. 检测正向/负向信号
2. 在满足条件时终止
3. 生成总结确认

### 验证点
- [ ] 正向信号 ≥ 4 时提议终止
- [ ] 负向信号 > 0 时继续
- [ ] 生成确认话术

---

## TC-INT-004: 快速模式测试

**类型**: functional
**优先级**: P1

### 输入
```
/interview --mode feature --quick
```

### 预期行为
1. 最多 3 轮问题
2. 精选 3 个盲点问题
3. 达到 3 轮强制终止

### 验证点
- [ ] 不超过 3 轮
- [ ] 生成简化计划

---

## TC-INT-005: 尽调输入测试

**类型**: integration
**优先级**: P1

### 输入
```
/interview --mode feature --input .claude/plans/due-diligence-xxx.md
```

### 预期行为
1. 读取尽调报告
2. 基于现有方案提问
3. 输出包含方案选择

### 验证点
- [ ] 正确读取尽调报告
- [ ] 问题包含方案对比
- [ ] 输出包含选择倾向
